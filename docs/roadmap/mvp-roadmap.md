# MVP Roadmap

## Phase 1: Repo Foundation
- create folder structure
- add docs
- add backend skeleton
- add sample threat event files
- freeze pre-exam repo

## Phase 2: Backend Core
- build event normalizer
- build anomaly rules
- build risk score engine
- build response engine
- add API routes

## Phase 3: AWS Integration
- connect Lambda handlers
- connect EventBridge
- connect DynamoDB
- connect SNS alerts
- add WAF blocking logic

## Phase 4: Demo Layer
- create simulation route
- generate mock threat events
- capture screenshots
- record demo walkthrough

## Phase 5: GitHub Polish
- architecture diagram
- improved README
- setup guide
- demo section
- future roadmap

## Success Criteria
The MVP is successful when it can:
- receive a sample event
- normalize it
- score it
- decide on a response
- expose results through an API or demo flow
